Разработать консольное приложение, в котором в качестве аргумента командной строки
Пример: ./prog [число], где число - необязательный аргумент, который может отсутствовать
подается число типа uint64_t
Если аргумент отсутствует при запуске программы, то на его месте будет рандомное число также размером в 8 байт, беззнаковое
Выводим двоичное представление этого числа (необходимо вывести все разряды числа, байты числа разделяются пробелами)
Выполняем преобразование в соответствие с вариантом 
Выводим на экран двоичное представление результата преобразования

Нужно обрабатывать ошибки ввода
Если например пользователь вместо ввода десятичного числа ввел строку, то сообщить об ошибке 
Или например запись вида 220v, тоже сообщить об ошибке
Рандомное число, в случае отсутствия аргумента, получать с помощью функций srand() и rand()


План:
1) Сделаем так, чтобы пользователь мог при запуске программы передавать число
Будем проверять, что было введено именно число, а не что либо другое
И заносить это число в переменную типа uint64_t
Если не было ничего введено, то в переменной будет рандомное число размером 8 байт беззнаковое

2) Выведем полученное число в двоичной системе 
Напишем алгоритм преобразования исходного числа 
Выведем новое число в двоичной системе 

Важные замечания:
1. Преобразование должно осуществляться только с помощью арифметических, логических и побитовых операций.
Использование массивов запрещено.

2. Использовать форматный спецификатор %b для вывода чисел в двоичной системе запрещено.

3. При ошибках ввода программа должна выдавать диагностическое сообщение на консоль, прежде чем завершиться.

4. Исходное число должно измениться 

5. Если операция предусматривает использование одного или нескольких случайных параметров (например, сдвиг на случайное число битов), их значения также следует выводить на экран.

6. Для преобразования строки в число рекомендуется использовать функцию sscanf() с правильно выбранным спецификатором формата и проверкой возвращаемого значения этой функции и переменной errno
